# Window Centering Next

窗口居中工具 — 基于 [Window Centering Helper](https://github.com/KamilSzymborski/kamilszymborski.github.io) 的增强分支，专为 Windows 多屏环境优化。

---

## 功能概览

### 双模式居中服务

| 模式 | 说明 |
|------|------|
| **自动居中** | 监听新窗口的创建，窗口出现后自动居中到屏幕中央 |
| **快捷键居中** | 通过预设的按键序列（如连按 3 次 `Left Shift`）手动触发当前窗口居中 |

两种模式可独立开关，灵活组合使用。

---

### 双屏幕 / 多显示器分别居中

**这是本分支最重要的改进。** 原始项目仅支持在 **主显示器** 上居中所有窗口。本分支新增了"在当前屏幕居中"的能力：

- **主显示器居中**（开关开启）: 无论窗口位于哪个屏幕，始终将其移动到主显示器中央
- **当前屏幕居中**（开关关闭）: 窗口会居中到它 **当前所在屏幕** 的中央

这意味着在多显示器环境下，窗口在副屏打开时会被居中在副屏中央，而非被"拉回"主屏。真正实现了双屏幕甚至多屏幕的分别居中。

---

### 自定义窗口尺寸

| 功能 | 说明 |
|------|------|
| **自定义窗口宽度** | 以屏幕宽度的百分比（2%–100%）调整窗口宽度 |
| **自定义窗口高度** | 以屏幕高度的百分比（2%–100%）调整窗口高度 |

> 滑块颜色会随数值变化提示：
> - 红色：不推荐范围（≤40% 或 100%）
> - 黄色：需注意范围（≤60% 或 98%）
> - 系统色：推荐范围

---

### 智能窗口过滤

| 选项 | 说明 |
|------|------|
| **仅新窗口** | 已居中过的窗口不会被重复处理 |
| **智能判断应排除的窗口** | 自动跳过最大化窗口、系统弹出窗口、通知区域图标等不适合居中的窗口 |
| **强制调整不可缩放窗口** | 对标记为不可调整大小的窗口强制执行尺寸变更 |

---

### 通用设置

| 功能 | 说明 |
|------|------|
| **语言** | 支持 **English** 和 **简体中文**，运行时可切换 |
| **开机自启** | 登录系统后自动在后台启动，驻留系统托盘 |
| **系统托盘图标** | 左键点击托盘图标打开设置面板，所有配置实时生效 |

---

### 快捷键居中可调参数

| 参数 | 说明 |
|------|------|
| **按键** | 可自定义触发按键（默认 Left Shift） |
| **次数** | 需要连按的次数（默认 3 次） |
| **超时** | 按键序列的超时窗口（默认 1050 ms） |

---

## 技术细节

- **框架**: .NET Framework 4.8 / Windows Forms
- **UI 库**: MetroFramework (Modern UI)
- **全局热键**: Gma.System.MouseKeyHook
- **窗口操作**: 通过 Windows API (`user32.dll` / `uxtheme.dll`) 直接操控窗口位置与样式
- **DPI 感知**: 启动时调用 `SetProcessDPIAware()` 确保高 DPI 屏幕下居中和尺寸计算准确

---

## 与原始项目的主要差异

1. **项目重命名**: `Window Centering Helper` → `Window Centering Next`
2. **双屏分别居中**: 新增"在当前屏幕居中"选项（关闭 `Center on primary screen` 开关）
3. **简体中文完善**: 补充了原项目中缺失的部分中文翻译，包括开机自启等 UI 文本
4. **开机自启**: 新增 Windows 启动时自动运行功能，带系统托盘常驻支持
